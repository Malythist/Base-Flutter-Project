# Инициализация приложения и Dependency Injection

Этот каталог содержит логику инициализации приложения.

## Ответственность

Слой app отвечает за:
- Запуск приложения
- Инициализацию Dependency Injection
- Подготовку глобальной конфигурации до отображения первого экрана

## Dependency Injection

В проекте используются:
- get_it как сервис-локатор
- injectable для генерации и регистрации зависимостей

Все зависимости регистрируются на старте приложения.

Типичные задачи:
- Регистрация Dio и сетевых сервисов
- Предоставление ApiConfig и конфигурации окружения
- Регистрация репозиториев, датасорсов и use case’ов

## Правила инициализации

- Нельзя создавать зависимости вручную внутри фич
- Все зависимости должны приходить из DI
- DI инициализируется до создания корневого виджета

Это гарантирует предсказуемый жизненный цикл и тестируемость.
